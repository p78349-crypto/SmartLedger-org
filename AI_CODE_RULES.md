# AI 코드작업 규칙 (프로젝트 규칙)

우선순위 및 규칙 (사용자 지시 기준):

1) 필수(우선순위 1) — 코드 작성 금지: 한 줄은 절대 80자를 초과하지 않는다.
   - 모든 Dart/코드 파일의 새/수정된 모든 라인은 80자 이내여야 한다.
   - 자동 포맷 전후에도 라인 길이를 검사하여 위반 시 수정한다.

2) 필수(우선순위 2) — 불필요 작업 금지: 사용자가 명시적으로 지시하지 않은 변경을
   수행하지 않는다.
   - 자동으로 파일을 생성/삭제/수정하지 않는다. 사용자 지시가 있을 때만 수행.
   - 대규모 리팩터, 기능 추가, 또는 의도하지 않은 API 변경은 사전 승인 필요.

3) 권장사항(제안): 아래 항목은 기본 동작이 아닌 권장 규칙이다. 필요 시 사용자에게
   제안하고, 승인을 받은 경우에만 적용한다.
   - 변경은 작은 단위의 커밋(단일 목적, 최소 변경)으로 수행.
   - 변경 후 `dart format` 및 `dart analyze` 또는 해당 언어의 포맷/정적분석을 실행.
   - 민감 정보(키/암호 등)는 절대 저장소에 추가하지 않음.
   - 변경 전 간단한 요약(무엇을, 왜, 예상 영향)을 제공.
   - 다단계 작업은 TODO로 분해하고 단계별로 보고.

적용 절차(권장):
- 새 변경을 시작하기 전에 사용자에게 한 줄 요약(무엇을 할지)을 보냄.
- 변경을 수행할 때 `AI_CODE_RULES.md`의 1~2 우선순위를 반드시 지킴.
- 변경 완료 후 `dart analyze`(또는 해당 도구) 실행 결과를 보고.

예외: 보안상 긴급 대응(예: 보안 키 유출 차단 등) 시에는 즉시 수정할 수 있으나,
사후에 사용자에게 보고하고 변경을 설명한다.

문의: 이 규칙을 수정하거나 확장하려면 구체적인 항목을 알려주십시오.
