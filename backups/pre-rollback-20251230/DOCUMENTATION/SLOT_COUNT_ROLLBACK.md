롤백 및 긴급대응 절차

1) 롤백 조건
- 변경 후 UI/데이터 손상(예: 아이콘 위치 소실, 페이지 인덱스 오류)이 확인되면 즉시 롤백을 검토합니다.

2) 롤백 방법
- Git: 변경된 커밋을 리버트하거나 `main`/`release` 브랜치로 되돌립니다.
  - 예: `git revert <commit>` 또는 `git checkout <release-tag>`
- 빠른 핫픽스: 문제 원인 파일을 식별해 중앙 상수 대신 이전 코드를 임시 복원하고 배포합니다.

3) 데이터 복구
- 사용자 prefs/DB가 슬롯 수 변경으로 손상된 경우, 마이그레이션 스크립트를 준비해 복원합니다.
  - 예: prefs에 저장된 슬롯 배열을 길이 `Page1BottomQuickIcons.slotCount`로 패딩하거나 자르기

4) 테스트 및 검증
- 롤백 후 전체 `flutter test`와 수동 UI 체크(핸드폰/에뮬레이터)를 실행하여 정상 동작 확인

5) 보고
- 롤백 시행 시 원인 분석 리포트를 작성해 팀 채널과 릴리스 노트에 첨부합니다.
