# UI 상태 정의 (자산/세무/문서)

## 공통 상태 원칙
- 로딩: Skeleton(카드/리스트/차트), 최대 1초 후 스피너 전환. 버튼은 disabled + progress.
- 빈 상태: 핵심 메시지 + 부가 설명 + 1차 CTA(추가/동기화/업로드) + 2차 CTA(가이드 보기).
- 오류: 제목+원인 요약, 재시도 버튼, 네트워크/권한 구분. 로그/신고 진입 링크.
- 권한 거부: 권한 요청 안내 + 시스템 설정 열기 버튼.
- 페이징/무한 스크롤: 하단 mini spinner, 에러 시 inline 재시도.

## 자산 현황
- 로딩: 총자산 카드 skeleton, 6개 그리드 skeleton 카드, 필터 skeleton chip.
- 빈: "등록된 자산이 없습니다" + CTA: 자산 추가, 파일 불러오기.
- 오류: "자산을 불러오지 못했습니다" + 재시도 + 백업에서 복원 CTA.
- 부분 데이터: 일부 자산군만 로드 시 각 카드에 워닝 배지.

## 자산 상세(확장)
- 로딩: 상단 카드 skeleton(5~6), 차트 placeholder, 리스트 skeleton 5줄.
- 빈: "거래가 없습니다" + CTA: 거래 추가/입금/충전.
- 오류: "거래를 불러오지 못했습니다" + 재시도.
- 차트 데이터 없음: 차트 영역에 빈 상태 일러스트 + 기간 변경 CTA.
- BottomSheet 로딩: 스피너 + 최소 메타만 표시.

## 자산 상세(간단)
- 로딩: Hero 카드 skeleton + 3건 리스트 skeleton.
- 빈: "거래가 없습니다" + CTA: 확장 상세 열기.

## 세금 계산/신고
- 로딩: 결과 카드 skeleton, 폼 입력은 즉시 표시(낮은 지연).
- 검증 실패: 필수 입력 미완성 시 CTA disabled + 오류 힌트.
- 결과 없음: 입력 부족 시 "입력 후 계산" 안내.
- 오류: "계산에 실패" + 재시도, 입력값 확인 안내.
- PDF 미리보기 로딩: inline spinner, 실패 시 "다시 시도".

## 리포트/절세
- 로딩: 히트맵 placeholder, 시나리오 카드 skeleton, 절세 카드 skeleton.
- 데이터 없음: "분석할 데이터가 부족" + 기간/자산 선택 CTA.
- 오류: 재시도 + 데이터 동기화 안내.

## 포트폴리오/배분
- 로딩: 도넛/바 skeleton, 리밸런싱 카드 skeleton.
- 데이터 없음: "포트폴리오가 설정되지 않았습니다" + 목표 비중 설정 CTA.
- 오류: "포트폴리오 계산 실패" + 재시도.

## 문서함
- 로딩: 파일 카드 skeleton 6~9개.
- 빈: "저장된 문서가 없습니다" + 업로드/스캔 CTA.
- 오류: "문서를 불러오지 못했습니다" + 재시도.
- 권한 없음(스토리지/카메라): 안내 + 권한 설정 버튼.
- 만료 임박: 배지 강조(경고색), 알림 토글 제안.

## 네트워크/오프라인 대응
- 오프라인 배너 + 로컬 캐시 사용 안내, 재연결 시 스낵바.
- 제출/업로드 시 큐잉 → 온라인 시 재전송.

## 접근성/언어
- 상태 메시지는 2줄 이하, 기본 폰트 크기 준수, 색상만으로 상태 구분 금지(아이콘+텍스트).
