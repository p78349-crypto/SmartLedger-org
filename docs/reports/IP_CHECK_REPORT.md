# 지적재산권 무단 사용 흔적 조사 보고서

## 개요
본 보고서는 `vccode1` 프로젝트 내에서 지적재산권(IP) 무단 사용 흔적을 조사한 결과를 요약합니다. 조사는 저작권 관련 키워드 검색, 라이선스 파일 확인, 의존성 패키지 검토, 그리고 코드 내 주석 분석을 통해 수행되었습니다.

## 조사 결과 요약

### 1. 저작권 및 라이선스 키워드 검색
- **검색 키워드**: `Copyright`, `License`, `All rights reserved`, `@author`, `copied from`, `based on`, `StackOverflow`
- **결과**:
    - `windows/runner/Runner.rc` 및 `macos/Runner/Configs/AppInfo.xcconfig` 등 플랫폼별 설정 파일에서 표준 Flutter 프로젝트 템플릿의 저작권 문구("Copyright (C) 2025 com.example. All rights reserved.")가 발견되었습니다. 이는 일반적인 사항입니다.
    - `lib` 폴더 내의 소스 코드(`.dart`)에서는 타사의 저작권 헤더나 라이선스 명시가 발견되지 않았습니다.
    - 일부 문서 파일(`DOCUMENTATION/SLOT_COUNT_DETAILED_LOG.md` 등)에서 "GitHub Copilot"이 작성자로 명시된 부분이 있으나, 이는 AI 도구 사용을 나타내는 것으로 문제가 되지 않습니다.

### 2. 의존성 패키지 검토 (`pubspec.yaml`)
- **검토 내용**: 프로젝트에서 사용 중인 외부 패키지 목록 확인.
- **주요 패키지**: `flutter`, `cupertino_icons`, `google_ml_kit`, `shared_preferences`, `provider`, `intl`, `drift` 등.
- **결과**: 확인된 모든 패키지는 Flutter 생태계에서 널리 사용되는 오픈 소스 패키지들로, 대부분 MIT, BSD, Apache 2.0 등 퍼미시브(Permissive) 라이선스를 따릅니다.
    - **권장 사항**: 앱 배포 시 오픈 소스 라이선스 고지 의무를 준수하기 위해, 앱 내 "오픈 소스 라이선스" 페이지를 제공하거나 `LicenseRegistry`를 통해 라이선스 정보를 노출하는 것이 좋습니다.

### 3. 코드 출처 및 주석 분석
- **검색 키워드**: `copied from`, `based on`, `stackoverflow.com`, `github.com` (소스 코드 내)
- **결과**:
    - 소스 코드 내에서 외부 코드를 무단으로 복사해왔음을 시사하는 주석(예: "// copied from stackoverflow...")은 발견되지 않았습니다.
    - `CMakeLists.txt` 파일 등에서 Flutter 프레임워크 이슈를 참조하는 GitHub 링크가 발견되었으나, 이는 프레임워크 차원의 주석으로 안전합니다.

### 4. 라이선스 파일 부재
- **특이 사항**: 프로젝트 루트 디렉토리에 `LICENSE` 파일이 존재하지 않습니다.
- **권장 사항**: 프로젝트의 배포 계획에 따라 적절한 라이선스 파일(예: MIT, Apache 2.0, 또는 독점 라이선스)을 루트 디렉토리에 추가하는 것을 권장합니다.

## 결론
현재 코드베이스에서 명백한 지적재산권 침해나 무단 코드 도용의 흔적은 발견되지 않았습니다. 사용된 외부 라이브러리들은 표준적인 오픈 소스 패키지들입니다. 다만, 프로젝트 자체의 라이선스 파일이 누락되어 있으므로 이를 보완하고, 앱 출시 시 사용된 오픈 소스 라이브러리들의 라이선스 고지를 포함하는 절차를 진행하시기 바랍니다.

## 조치 완료 사항 (2025-12-25)
1. **LICENSE 파일 추가**: 프로젝트 루트에 MIT License 기반의 `LICENSE` 파일을 생성하였습니다.
2. **오픈 소스 라이선스 고지 기능 구현**: `SettingsScreen` (설정 화면) 하단에 "오픈소스 라이선스" 메뉴를 추가하여, 앱에서 사용 중인 패키지들의 라이선스 정보를 확인할 수 있도록 구현하였습니다 (`showLicensePage` 사용).

새로운 기능을 추가하거나 외부 라이브러리를 도입할 때 아래 항목을 점검하고 위 로그에 기록하십시오.

1. **새로운 패키지 추가 시**
   - [ ] `pubspec.yaml`에 추가된 패키지의 라이선스 확인 (MIT, BSD, Apache 등 상용 이용 가능 여부)
   - [ ] GPL 계열(GPL, AGPL) 라이브러리가 포함되지 않았는지 주의 (소스 코드 공개 의무 발생 가능성)
   - [ ] 앱 내 "오픈소스 라이선스" 페이지에 자동 반영되는지 확인 (`flutter pub get` 후 확인)

2. **외부 코드 참조 시**
   - [ ] StackOverflow, GitHub 등에서 코드를 복사해올 경우, 출처와 라이선스 주석 표기
   - [ ] "Copied from", "Based on" 등의 주석이 코드에 남아있는지 확인하고, 필요시 리팩토링 또는 출처 명시

3. **리소스(이미지, 폰트) 추가 시**

   - [ ] 저작권자 표기가 필요한 경우 앱 내 "정보" 또는 "라이선스" 페이지에 명시
